# 역기획서 — 방 기반 주거 (Room-based Housing)

> 비전: **"유저 개입이 거의 없이 알아서 번창하는 림월드."** 플레이어가 벽을 긋지 않는다.
> 콜로니가 *욕구*로 스스로 집을 짓고, 방을 나누고, 가구를 들인다.
> 두 기둥: 넓게 펼쳐라(spread-out-not-stack), 행동은 욕구에서 창발한다(needs-driven-emergence).

## 1. 림월드에서 가져오는 것 (역기획)

| 림월드 메커니즘 | 핵심 | 우리 적용 |
|---|---|---|
| 벽/문/바닥 타일 | 공간을 *둘러싸야* 방이 된다 | `Wall`/`Door`/`Floor` 타일 추가 |
| 방 = 폐쇄 공간 | flood-fill로 벽·문에 막힌 영역 탐지 | M2에서 방 감지 도입(M1은 건축으로 폐쇄 보장) |
| 문 | 여닫이, 통행 지연, 열어두면 누수 | M1: 통행비용↑. 후반: 개방 페널티(무드/보온/보안) |
| 가구 | 침대·저장·화덕·식탁이 행동을 *물리적으로* 앵커 | 침대(M1) → 저장구역·화덕·식탁(M2) |
| 방 역할 | 내용물로 침실/창고/주방/식당 분류 | M2 |
| 무드(멘탈) | 방 품질·기능 분리가 기분을 좌우 | M3, era/부유도로 가중치 점증 |

**우리만의 난점:** 림월드는 플레이어가 짓는다. 우리는 콜로니가 *효용 중재기*로 "이제 침실을 떼자"를 스스로 결정해야 한다. 즉 모든 건축은 욕구 산출물이다.

## 2. 엔진 적합성 (왜 새 프로젝트가 아니라 변형인가)

- 타일 월드 + 4방향 A*가 **per-tile `moveCost`** 기반 → `Wall=∞`, `Door=비용↑`, `Floor=보행`만 추가하면 경로탐색이 자동 대응.
- 타일은 1글자 코드로 직렬화 → **새 타일 타입은 저장이 공짜**(`TILE_CODES`에 등록만).
- 물리적 아이템 레이어(운반)·욕구 브레인·렌더·UI·낮밤·창발 도로는 그대로 재사용.
- 변경 격리: **`kind === "house"`만** 벽 방으로 바꾸고, 공용 건물(창고/부엌 등)은 당분간 기존 솔리드 블록 유지 → 한 번에 무너지지 않게.

## 3. 마일스톤

### M1 — 벽·문·집 내부·침대
- **M1a (이번 슬라이스):** 타일 `Wall`/`Floor`/`Door`. 집 = 3×3(둘레 벽, 내부 1칸 바닥, 도로변 문 1개). `home`=내부 바닥칸. 주민이 문을 통해 **집 안으로 들어가 바닥에서 잔다**(노숙보다 회복↑). 렌더·저장.
- **M1b:** 침대(가구). 수면 우선순위 **침대 > 집안 바닥 > 노숙**. 침대는 욕구(쾌적/회복)로 주민이 직접 제작·배치. 침대 수면 회복 최고.
- 수용 기준: 1인 세계에서 집이 벽으로 둘러싸이고 문으로 드나들며, 침대를 만들면 침대에서 잔다. 경로탐색이 벽을 막고 문으로 통한다. 저장/로드 유지.

### M2 — 구역화(zoning)와 방 감지
- flood-fill 방 감지(벽/문 경계). 가구/구역: 저장구역(창고를 방 안으로), 화덕(요리), 식탁(식사).
- **기능 합방 페널티:** 침실+주방+창고가 한 방이면 약한 불편(초반엔 미미). 분리하면 이점.
- 집터 확장 + 내부 칸막이(벽 추가)로 방 쪼개기 — 욕구로 발현.

### M3 — 멘탈(무드)
- 무드 = 방 품질(넓이/분리/청결/장식) + 사건. **가중치는 era/부유도로 점증**(초반 생존, 후반 삶의 질).
- 낮은 무드 → 능률↓·행동 일탈. 높은 무드 → 보너스. 무드가 "방 쪼개기/확장" 욕구를 끌어낸다.

## 4. M1a 구현 노트 (이번)
- 타일 추가: `Wall`(∞), `Floor`(보행 1.0), `Door`(보행 1.3, 여닫이 마찰 표현). `TILE_CODES`: `#`/`.`/`+`.
- 집 크기 2×2 → **3×3**(`findHouseSite`, `registerBuilding`, 도어 계산, 재개발/수용량 영향 확인).
- 완공 시: 둘레=Wall, 도로변 중앙=Door, 내부=Floor. `agent.home`=내부칸. 노숙 대신 실내 수면.
- 렌더: 집은 솔리드 블록 대신 타일(벽 2.5D 융기, 바닥 톤, 문 틈)로 그림. 공용 건물은 기존 유지.
- 후방 호환: `House`/`HouseSite`/`HouseFoundation` 타입 유지(공용 건물·구 세이브).

## 5. 결정 사항 (확정)
- **벽 토폴로지: 타일형(림월드식).** 엣지형 검토했으나 "최대한 림월드스럽게"를 우선. 셀 차지는 ① 방을 키우고 ② 벽 오토타일로 완화.
- **시작 집: 4×4(내부 2×2 = 4칸).** 초기엔 침대·창고·요리·식사를 한 방에 욱여넣는 크기. 내부 분할/확장은 M2.
- **벽 오토타일:** 벽 셀이 이웃(벽/문)을 보고 노출된 모서리에만 음영/하이라이트를 그려 하나의 연결된 벽 덩어리로 보이게(코너=L). 림월드식.
- **벽·문·바닥은 선택 가능한 구조물 오브젝트:** 집 푸트프린트 안이라도 클릭하면 건물이 아니라 그 타일(벽/문/바닥)을 인스펙트.
- 문: M1은 자동 통과(비용만), 개방 상태/페널티는 M3 무드와 함께.
- 무드 점증: 실시간이 아닌 **era/부유도 연동**(결정적·의미적).

**Stage 2 "벽 공유 증축" 완료 (이번 세션):**
공용 주택을 공유하는 주민이 자재 여유가 생기면 **개인 침실(3×3, 내부 1칸)을 한 칸 증축**한다. 기존 벽 한 줄을 공유(이중벽 X)하고, 그 공유벽에 **내부 문 한 칸**을 뚫어 공용방으로만 통한다.
- 새 건물 종류 `"bedroom"`(`ROOM_BUILDING_KINDS`에 포함, `Building.annexOf`로 부모 집과 연결). 집 점유/수용량 enumeration은 `kind === "house"`라 침실은 자동 제외 — 수용량/거주자 수에 안 잡힘.
- 배치: `Simulation.findAnnexSite(parent)` — 4면×오프셋을 훑어, 공유 엣지가 **실제 부모 벽(Wall)** 위에 있고 문의 부모쪽 이웃이 **내부 Floor**인 자리를 찾음(없으면 undefined). 신규 타일은 잔디·미점유여야.
- 내부 문: `registerBuilding`에 `doors`/`annexOf` 명시 인자 추가(외벽/도로용 `computeDoors` 우회). 침실은 `setBuildingStage`에서 도로 예약/포장 스킵, 푸트프린트 페인트가 **기존 Wall/Door를 덮지 않음**(공유벽 보존).
- 자재 절약: `ensureBuildPlan`이 **이미 목표 타일과 같은 칸은 done 처리** → 공유벽은 다시 안 깖/안 냄, 문(공유벽 Wall→Door)과 신규 구조만 시공.
- 발현/소유권: `AgentBrain.tryBuildPrivateRoom`(doProductiveWork, 침대·식탁 다음). 조건 = 공용 주택 거주(occupants≥2) + 창고 wood ≥ `PRIVATE_ROOM_WOOD_SURPLUS(40)` + 본인 침실 미보유. 기초 자재(`FOUNDATION_WOOD`) 손에 없으면 먼저 fetchWood. 등록 즉시 foundation까지 깔고 BuildHouse로 — "site"에 머물러 부모-도로 문으로 라우팅되는 버그 회피. 완공 시 `finishBuilding`의 bedroom 분기가 owner 지정 + `relocateBedInto`로 침대를 공용방→개인방 이전(homeBuildingId는 공용집 유지).
- 검증 완료(preview 스크립트): 공유벽 1줄·내부 문·이중벽 없음, 침실은 공용집을 통해서만 도달(7스텝), 수용량/거주자 불변, save/load 라운드트립(kind·annexOf·문 타일), 실제 brain 35스텝 만에 자가 증축 + 침대 이전. tsc/build 클린.
