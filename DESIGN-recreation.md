# 역기획서 — 놀이공원 / 놀이기구 (멘탈 회복 여가 시설)

작성: 2026-06-19. 원래 요청 #4 — "멘탈보호용 컨텐츠, 놀이기구(롤러코스터·바이킹 같은), 생각보다 고퀄리티".

## 1. 왜 / 북극성

[[needs-driven-emergence]] — 마을이 성숙할수록 **삶의 질**이 중요해진다. 생존(식량·집)이 충족된 마을이
다음으로 갖춰야 할 것이 **여가·멘탈 회복** 시설이다. 놀이기구는 그 첫 번째 "고품질 여가 컨텐츠"로,
주민이 스스로 찾아가 즐기고 기분이 좋아지는 장소다.

현재 공백 (이 기획이 메우는 것):
- **leisure(여가)** 욕구는 지금 `wanderNearHome`(배회)로만 채워짐 → 목적지/볼거리 없음.
- **공원**(comfort 회복 장소)은 #1에서 시민 건물 정리 후 자동 건설에서 빠짐 → comfort 능동 회복 장소 부재.
- → 놀이공원이 **여가 + 무드(멘탈)** 를 회복하는 매력적인 목적지가 된다 (공원의 상위 호환).

## 2. 시설 — 놀이공원(`funfair`), 첫 놀이기구 = **롤러코스터** (확정)

- **`funfair`(놀이공원)**: 야외 시설 (벽으로 둘러싼 방이 아님 — 공원·목초지처럼 자체 타일/장식으로 그림).
  - 푸트프린트 ~8×6 광장(코스터 루프를 담을 크기). 가장자리 울타리/입구, 포장 바닥, 탑승 플랫폼, 그리고 **놀이기구**.
  - 확장 가능한 설계: `funfair`가 "기구 종류(rideType)"를 가지며, 1단계는 **롤러코스터(coaster)**.
    이후 바이킹·회전목마를 같은 틀로 추가.
- **롤러코스터 (확정)**: 닫힌 **트랙 루프**(큰 리프트 언덕 + 낙하 + 범프) 위를 **차량(들)이 질주**.
  - **트랙은 타일이 아니라 곡선**: 건물 원점 기준 픽셀 좌표의 점열(닫힌 루프)로 정의하고 트랙·지지대를
    건물/장식 레이어에 그린다 (트랙 타일 불필요 → 경로탐색·세이브에 영향 없음).
  - **차량 애니메이션**: 매개변수 `s∈[0,1)`로 트랙을 따라 이동(`pos = track(s)`), 차체는 국소 접선 방향으로
    기울임. 리프트에선 천천히, 낙하에선 빠르게(`ds/dt`를 트랙 높이로 변조)해 실제 코스터 느낌.
    **동적 오버레이**(분수·기차처럼 매 프레임)에서 시간 기반으로 그림.
  - 탑승객 = 차량 안 작은 실루엣(낙하 구간에서 팔 든 모습 같은 디테일은 여유 되면).
  - "고퀄": 컬러풀한 차량, 비계(scaffold) 지지대와 그림자, 곡선 트랙 음영, 속도감.

대안 기구(향후, 같은 `funfair` 틀): 바이킹(진자 스윙), 회전목마(회전).

## 3. 동작 (needs-driven, 발현형)

- **leisure 욕구가 높을 때** 주민이 놀이공원으로 가서 탑승 → 여가가 크게 차고 **무드 보너스**.
  - 새 상태: `MoveToFunfair` → `Ride`. (지금 leisure는 `wanderNearHome`만 → 놀이공원이 있으면 그쪽을 우선.)
- **탑승 효과** (식탁의 `SEATED_MEAL_*` 보너스를 본보기로):
  - `leisure` 대량 회복 (배회보다 훨씬 큼) — 그래서 주민이 배회보다 놀이공원을 선호.
  - `mood` 즉시 상승 (기분 좋은 경험).
  - **멘탈 보호**: 최근 탑승한 주민은 한동안 mood break(우울 일탈)에 더 강함 (선택: `agent.funAt` 타임스탬프로).
- **수용량 / 사이클**: 배가 한 번에 N명(예 4~6) 태움. 한 사이클 ~수 초 흔들린 뒤 하차(기분↑), 다음 대기자 탑승.
  대기 줄(선택, 후순위) — 1단계는 빈 좌석 있으면 타고, 없으면 잠깐 기다리거나 배회.

## 4. 경제 / 건설

- **건설 조건**: 사치재 → era ≥ 2(읍내 이상) + 인구 임계(예 ≥ 12) + **자재 여유**가 있을 때만.
  생존 시설(창고·식량창고·부엌)이 먼저, 그 다음 여가.
- **자재**: 나무 + (고퀄/대형이므로) 돌, 후기엔 강철까지 — "재료 숙달" 방향([[materials-mining-direction]])과 연결.
- **수요 비례 증설**: #3에서 만든 `desiredCivicCount`/`needsMoreOf` 틀을 재사용 — 1 per ~24명 등.
  (단, 야외 시설이라 piecemeal 벽쌓기 대신 공원/목초지식 완성 흐름을 따름.)

## 5. 렌더링

- 야외 시설: `drawBuilding`의 park/pasture 분기처럼 `funfair` 전용 그리기 — 광장 바닥, 울타리/입구,
  지지대·플랫폼, 장식.
- **애니메이션**: 동적 오버레이 레이어(매 프레임 clear)에서 시간 t로 배 스윙을 그림. 시간원은 렌더 루프의
  경과시간(분수/등불/밤 연출이 쓰는 것과 동일). 탑승객은 작은 점/실루엣.
- 인스펙터: `funfair`(놀이공원) 이름 + 탑승 인원/상태. HUD는 변경 없음.

## 6. 통합 지점 (구현 시 손댈 곳)

- `types.ts`: `BuildingKind += "funfair"`; (야외라 ROOM_BUILDING_KINDS에는 **안** 넣음); `AgentState += MoveToFunfair, Ride`;
  `Agent.funAt?`(멘탈 보호용, transient).
- `Simulation.ts`: `getFunfair`/`nearestFunfair`; 완성 시 광장/지지대 타일 페인트; 탑승 좌석/사이클 상태;
  `desiredCivicCount`에 funfair 추가(증설). 새 타일이 필요하면(예 RideSupport/Platform) MOVE_COSTS·TILE_CODES 추가.
- `AgentBrain.ts`: leisure 드라이브가 `funfair`가 있으면 `goRide`(MoveToFunfair→Ride)로 가도록; `ride()`가
  leisure·mood 회복; `communalProject`에 funfair 건설 트리거(era≥2 + 인구 + 자재 여유); 멘탈 보호 적용.
- `PixiRenderer.ts`: `drawFunfair`(정적) + 오버레이의 바이킹 스윙 애니메이션 + 탑승객.
- `Inspector.tsx`/`App.tsx`: 이름 "놀이공원".

## 7. 단계 (차곡차곡, 검증하며)

- **R1 — 시설 + 기본 동작**: `funfair` 건물(야외 광장 + 정적 바이킹 그림) 건설; 주민이 찾아가 `Ride` →
  leisure·mood 회복. 검증: 여가 낮은 주민이 놀이공원으로 가서 기분이 오른다.
- **R2 — 애니메이션**: 바이킹 배 진자 스윙 + 탑승객 표시 (고퀄 패스). 검증: 스윙이 부드럽게 보인다.
- **R3 — 멘탈 보호 + 수요 증설**: 최근 탑승 시 mood break 저항; 인구 비례로 추가 건설; (선택) 대기 줄,
  2번째 기구(회전목마).

## 8. 확정 사항 (사용자 결정, 2026-06-19)

1. **첫 기구 = 롤러코스터** (가장 화려한 쇼피스).
2. **주 효과 = 여가 + 무드 + 멘탈보호** (최근 탑승 주민은 우울 일탈에 한동안 저항).
3. 진행 방식: **R1 → R2 → R3 단계별 검증·배포** (이 프로젝트 방식대로).

## 9. 현황 갱신 (2026-06-20) — 소형 코스터 가동 + 대형은 엔딩으로

- **현재 사용**: 놀이공원 부지(8×6) **안에 들어가는 소형 평면 코스터**. 게임의 레일 스타일(받침·침목·강철 레일)로 납작하게 그리고, 차량 4량이 작은 루프를 돈다. 주민이 역으로 와서 **실제로 차량에 탑승**해 한 바퀴 돌고(위치가 루프를 따라 이동) 역에서 내리며 여가+무드+멘탈보호를 얻는다. 트랙/열차칸 위치는 `Simulation.coasterTrackTiles/coasterCarTiles`(타일 좌표)가 소유하고 렌더러·탑승 주민이 공유.
- **엔딩 콘텐츠로 보존(대형)**: 맵 전체를 굽이치는 **대형 코스터**(나중에 모든 자재를 모아 건설). 두 가지 시도를 했고 코드 히스토리에 있음:
  - 입체(2.5D) 버전 — 보라 기둥 위 공중 트랙, 높이별 굵기(원근), 수직 루프·헬릭스. (탑뷰 도트 배경엔 과해서 보류)
  - 평면 맵-전체 버전 — 아래 정규화 waypoint를 맵 크기에 곱해 Catmull-Rom으로 부드럽게.
  - 대형 평면 waypoint(부활용): [0.5,0.86],[0.76,0.85],[0.89,0.71],[0.9,0.5],[0.78,0.4],[0.62,0.49],[0.66,0.64],[0.54,0.7],[0.45,0.6],[0.5,0.45],[0.62,0.33],[0.76,0.22],[0.6,0.13],[0.4,0.15],[0.23,0.24],[0.12,0.46],[0.17,0.69],[0.31,0.82] (닫힌 루프).
- **엔딩 트리거(향후)**: 자재(나무/돌/강철 등) 대량 비축 + 인구 임계에서 "대형 롤러코스터" 건설 → 위 대형 트랙으로 전환, 더 많은 정원/대기줄.
